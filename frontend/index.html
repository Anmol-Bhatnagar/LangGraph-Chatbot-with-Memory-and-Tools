System Architecture

This chatbot demonstrates an advanced AI memory paradigm using LangGraph and SQLite:

  1. Short-Term Memory:

    Consists of the direct list of messages inside the LangGraph state. Capped to prevent context-window bloat and reduce costs.

  2. Long-Term Memory:

    A local SQLite database storing facts extracted by the LLM. Injected into the System Prompt on each interaction.

  3. Dynamic Extraction Nodes:
    • Turn Analyzer: Runs after every message turn. Decides whether to extract user facts or preferences to save in SQLite.
    • History Pruning: Cursors older messages when limit is reached. Summarizes their content into keypoints and commits to SQLite before removing them from state.
